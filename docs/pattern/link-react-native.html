APG pattern · react-native

Link

role: link

APG: https://www.w3.org/WAI/ARIA/apg/patterns/link/

About

A link widget provides an interactive reference to a resource. The target resource can be either external or local, i.e., either outside or within the current page or application.

ARIA roles

This pattern uses role link.

ARIA contract from aria-query

link
Accessible name required
Yes
Name from
author, contents
Supported properties
aria-disabled, aria-expanded, aria-haspopup, aria-atomic, aria-busy, aria-controls, aria-current, aria-describedby, aria-details, aria-dropeffect, aria-flowto, aria-grabbed, aria-hidden, aria-keyshortcuts, aria-label, aria-labelledby, aria-live, aria-owns, aria-relevant, aria-roledescription
Spec
https://www.w3.org/TR/wai-aria-1.2/#link

Keyboard interactions from APG

KeyAction
EnterExecutes the link and moves focus to the link target.
Shift + F10 (Optional)Opens a context menu for the link.

Elements that natively carry one of this pattern's ARIA roles. Using these gives you the role for free — no role="…" attribute needed.

  • <area[href]> · implicit role link
